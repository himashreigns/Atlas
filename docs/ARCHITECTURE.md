## Performance Metrics

```
Theoretical Peak Performance:
  MAC_ops/cycle = N (MAC array width)
  GOPS = N Г— Frequency (GHz)
  
  Example: N=8, F=200MHz
  вҶ’ 1.6 GOPS

Actual Performance (with overhead):
  Efficiency вүҲ 60-80%
  вҶ’ ~1.0-1.3 GOPS actual

Latency per Inference:
  T = (Total_MACs / N) + Overhead
  
  Example: 28Г—28 CONV with 3Г—3 kernel, 6 filters
  MACs = 28Г—28Г—6Г—9 = 42,336
  Cycles = 42,336/8 + 500 вүҲ 5,800 cycles
  Time = 5,800 / 200MHz = 29 Вөs
```

## Key Design Decisions

1. **Streaming vs. Batch Processing**
   - Chosen: Streaming with row buffers
   - Benefit: Minimal memory, continuous flow
   - Trade-off: More complex control

2. **Fixed vs. Configurable**
   - Chosen: Fixed parameters at design time
   - Benefit: Optimized synthesis, simpler control
   - Trade-off: Less flexible, need recompile for changes

3. **Systolic vs. Other Dataflows**
   - Chosen: Systolic array with N stages
   - Benefit: Regular structure, good utilization
   - Trade-off: Pipeline latency

4. **Memory Hierarchy**
   - External: DRAM for weights/data
   - On-chip: BRAM for active data
   - Registers: MAC array pipeline
   - Benefit: Balanced bandwidth/capacity

This architecture achieves:
вң“ Low latency inference
вң“ Minimal memory footprint
вң“ High MAC utilization
вң“ Scalable to different network sizes
